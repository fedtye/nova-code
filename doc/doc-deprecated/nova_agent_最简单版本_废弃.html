Nova (Converged)
产品定位 产品哲学 核心能力 工作流 扩展系统

高信任开发协作者

Nova 是运行在终端中的 AI 开发协作者。 它不追求自治、不模拟组织结构，而是帮助开发者更快、更稳地完成软件工程任务。

产品定位

Nova 不是 AI 操作系统，也不是自治 Agent 平台。 它是一个面向代码工作流的高信任执行协作者。

产品哲学

Human-in-the-loop 优先

所有关键操作必须可预览、可解释、可中断。

强上下文理解

围绕代码库、Git 历史与工程结构构建长期理解能力。

单 Agent 优于多 Agent

避免复杂组织结构，提升可控性与可调试性。

执行优先，而非自治

AI 负责执行复杂步骤，而不是自主决策业务方向。

核心能力

1. 代码理解

跨文件、跨模块理解项目结构与依赖关系。

2. Plan-first 执行

执行前生成计划与 diff，用户确认后执行。

3. Diff 驱动编辑

所有修改均以 diff 形式呈现，可回滚。

4. 工具执行能力

Shell / Git / 文件系统工具统一调用接口。

5. 长上下文记忆

持续理解项目状态，而非短期对话。

工作流设计

标准执行流程

用户输入 → 意图理解 → 生成计划 → 展示 diff → 用户确认 → 执行 → 结果反馈

错误处理

失败时自动分析原因，但必须重新生成计划，不直接盲目重试。

关键原则

  • 所有危险操作必须确认
  • 所有文件修改必须可回滚
  • 所有执行步骤必须可解释

扩展系统

MCP 工具协议

通过标准协议接入外部工具与数据源。

Hooks 系统

支持在关键生命周期插入脚本，但不影响核心执行模型。

（已收敛）能力说明

不再提供：Agent 市场 / 自进化系统 / 多角色编排 / 消息网关系统
