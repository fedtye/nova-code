v1.0 Product Design

下一代智能 CLI Agent

融合 Claude Code 的深度理解、Codex CLI 的极速体验、Hermes Agent 的自我进化能力与 OpenCode 的开放生态， 打造面向广泛用户群体的命令行智能助手。不止于代码，连接一切本地与云端能力。

产品版本 v1.0 文档日期 2026/05/21 状态 设计中

01 产品定位

Nova Agent 是一款运行在终端环境的智能体（Agent）工具，以自然语言为核心交互方式， 深度集成本地文件系统、Shell 环境、版本控制与外部服务。不同于传统的代码补全工具或聊天机器人， Nova Agent 具备自主任务规划、多步执行、错误自修复、持续学习的能力， 能够在获得用户授权后独立完成从需求分析到代码交付的完整工作流。

🎯

自然语言驱动

用户通过日常语言描述需求，Agent 自动拆解为可执行步骤，无需记忆复杂命令或 API。

⚡

极速本地执行

核心逻辑运行在本地 Node.js 环境，文件操作和命令执行零延迟，保障数据隐私。

🔌

开放扩展生态

原生支持 MCP（Model Context Protocol）与 Hooks 系统，连接任意数据源与外部工具。

🧠

自我进化

通过 GEPA 循环（Generate-Evaluate-Prune-Archive）持续积累领域技能，越用越聪明。

🌐

多模型兼容（规划中）

v1.0 暂仅支持内置模型。架构层已完全解耦，后续版本将开放 300+ 模型接入与动态切换能力。

👥

广泛用户覆盖

从专业开发者到产品经理，从运维工程师到数据分析师，统一入口满足不同角色需求。

02 目标用户画像

产品设计上兼顾专业深度与入门友好，覆盖以下四类核心用户群体：

👨‍💻

全栈开发者

需要跨文件重构、自动化测试、CI/CD 集成，追求效率最大化。

📊

数据分析师

通过自然语言处理数据清洗、可视化、报表生成，降低 Python/SQL 门槛。

🎨

产品经理

快速生成原型、撰写需求文档、分析竞品页面，无需依赖开发资源。

🛡️

运维工程师

自动化日志分析、服务器巡检、故障排查脚本编写与执行。

03 核心功能 — 交互模式

3.1 多模态交互入口

  • ✓ REPL 会话模式 — 进入 nova 后持续对话，支持上下文记忆与多轮澄清
  • ✓ 单行命令模式 — nova -q "生成用户认证模块" 直接输出结果后退出
  • ✓ 管道模式 — cat error.log | nova "分析错误原因" 支持标准输入重定向
  • ✓ 文件批处理模式 — nova --batch *.md "翻译成英文" 批量处理文件
  • ✓ Web Dashboard — nova dashboard 启动本地 React 可视化面板，查看任务队列、Skill 图谱、模型性能，与终端状态实时同步
  • ✓ 消息网关 — 同一 Agent 状态在 Slack、Telegram、Discord 间实时同步，支持通过 Bot 接收任务指令与推送进度通知

3.2 智能输入补全

  • ✓ 命令补全 — 基于当前项目上下文，Tab 补全文件路径、Git 分支、自定义指令
  • ✓ 意图预测 — 输入前缀时实时推荐常见操作，如 "deploy" → "deploy to staging and run smoke tests"
  • ✓ 历史召回 — 模糊匹配历史会话，支持 /history search "数据库迁移"

3.3 视觉反馈系统

  • ✓ 流式输出 — 思考过程与最终结果分段显示，支持 Markdown 实时渲染
  • ✓ 差异预览 — 文件修改前展示语法高亮的 diff，确认后写入磁盘
  • ✓ 进度指示 — 多步任务显示进度条与当前步骤说明，支持 Ctrl+C 优雅中断
  • ✓ 状态徽章 — 终端提示符集成当前 Git 分支、Python 环境、Agent 状态等上下文徽章

04 核心功能 — 任务执行

4.1 自主任务规划

  • ✓ 目标拆解 — 接收高层需求后自动生成可执行子任务 DAG（有向无环图），识别依赖关系
  • ✓ 动态重规划 — 执行过程中遇到未预期错误时，自动分析并调整后续计划
  • ✓ 人机协作确认 — 关键操作（如删除文件、推送代码、执行高危命令）前主动请求用户确认
  • ✓ 持久化任务 — 支持 /goal 指令创建跨会话的长期目标，后台异步执行并推送进度

4.2 子代理编排

  • ✓ 并行子代理 — 同时启动多个 Specialist Agent（代码、测试、文档、审查），并行处理独立子任务
  • ✓ 代理间通信 — 子代理通过基于工具控制面的消息总线交换中间结果，所有通信封装为标准化工具调用，避免上下文冗余
  • ✓ 结果聚合 — 协调者代理（Orchestrator）合并多代理输出，解决冲突并生成统一结论
  • ✓ 代理市场 — 用户可安装社区共享的 Specialist Agent，如 "React 专家"、"SQL 优化器"

4.3 安全沙箱

  • ✓ 统一工具策略审查 — 所有外部操作（Shell 命令、文件读写、HTTP 请求、MCP 调用）均通过工具控制面执行，自动匹配策略规则：高危命令 AST 解析拦截、敏感路径保护、域名白名单、分级授权确认
  • ✓ 运行时隔离 — 子 Agent、MCP Server、Hooks 运行在独立进程，受 CPU/内存/超时资源限制，异常崩溃不影响主 Agent
  • ✓ 全链路审计 — 所有工具调用（命令、文件、网络、MCP）结构化记录：调用者、参数、结果、耗时，支持导出与回溯
  • ✓ 操作回滚 — 所有文件修改先写入临时副本，支持 /undo 一键回滚到任意历史状态

05 核心功能 — 代码编辑

5.1 智能代码生成

  • ✓ 多文件感知 — 编辑时自动检索项目中的类型定义、接口契约、测试用例，确保一致性
  • ✓ 上下文窗口优化 — 采用 RAG（检索增强生成）动态加载最相关的代码片段，突破模型上下文限制
  • ✓ 测试驱动 — 生成代码时同步生成单元测试，运行失败自动迭代修复
  • ✓ 风格继承 — 分析现有代码的 ESLint/Prettier 配置与命名约定，生成风格一致的代码

5.2 重构与审查

  • ✓ 跨文件重构 — 重命名符号时自动更新所有引用，支持提取组件、内联函数等复杂重构
  • ✓ 文档同步 — 修改函数签名后自动更新 JSDoc/TSDoc 与 README 中的相关说明
  • ✓ 遗留代码迁移 — 支持将 Python 2 迁移到 3、React Class 迁移到 Hooks、CommonJS 迁移到 ESM 等场景

5.3 代码审查引擎

  • ✓ 增量 diff 分析 — nova review 自动分析当前工作区变更，定位潜在 Bug、性能瓶颈与安全漏洞
  • ✓ 架构异味检测 — 识别圈复杂度过高、重复代码、违反 SOLID 原则等架构层面的问题
  • ✓ 规则可配置 — 支持自定义审查规则集，适配团队代码规范与合规要求
  • ✓ Git 工作流集成 — 在 PR / Merge Request 阶段自动触发审查，输出结构化报告并推送通知

06 核心功能 — 上下文管理

6.1 持久化记忆

  • ✓ 项目记忆 — 自动学习项目结构、技术栈、命名规范，跨会话保持上下文
  • ✓ 用户偏好 — 记忆用户的代码风格、常用命令、拒绝过的方案，后续自动适配
  • ✓ 对话压缩 — 长会话自动提取关键决策与结论，释放上下文窗口空间
  • ✓ 知识图谱 — 构建项目实体关系图（模块、接口、数据库表），支持图查询语言检索

6.2 上下文注入

  • ✓ @ 引用系统 — @file:src/auth.ts、@pr:#42、@doc:API.md 精确注入外部上下文
  • ✓ 截图理解 — 拖拽图片到终端，Agent 解析 UI 设计图并生成对应 HTML/CSS 代码
  • ✓ URL 抓取 — @url:https://api.example.com/docs 通过 http_request 工具安全抓取并解析外部文档，受域名白名单与审计策略约束
  • ✓ Git 集成 — 自动读取最近提交历史、分支差异、PR 描述作为决策依据

07 核心功能 — 扩展体系

7.1 MCP 协议支持

  • ✓ 原生 MCP Client — 内置 Model Context Protocol 客户端，连接任意 MCP Server
  • ✓ 工具发现 — 自动枚举 MCP Server 提供的工具列表，动态生成功能说明
  • ✓ MCP Server 市场 — 一键安装社区 MCP Server（数据库查询、浏览器控制、Slack 通知等），与 Skill 市场共同构成统一扩展市场
  • ✓ 安全隔离 — 每个 MCP Server 运行在独立进程，异常崩溃不影响主 Agent

7.2 Hooks 系统

  • ✓ 生命周期钩子 — before-command、after-file-change、on-error 等事件点自定义行为
  • ✓ 条件触发 — 基于正则匹配、文件类型、Git 状态等条件触发钩子
  • ✓ 脚本生态 — 钩子可配置 Shell 脚本、JavaScript 函数或通过工具控制面调用 HTTP 工具与外部服务，统一受审计与限流策略保护
  • ✓ 钩子市场 — 共享社区钩子模板，如 "提交前自动格式化"、"错误时发送钉钉通知"

7.3 Skill 系统

  • ✓ Skill 文档 — Agent 每执行 15 次工具调用后自动评估并生成 Skill Document，记录成功经验
  • ✓ Skill 市场 — nova skills add react-expert 安装预训练领域技能包，与 MCP Server 市场共同构成统一扩展市场
  • ✓ Skill 组合 — 多个 Skill 可组合为 Workflow，实现复杂领域任务的自动化
  • ✓ 私有化 Skill — 企业用户可训练内部 Skill 并托管在私有仓库，不与社区共享

09 SWOT 分析

S

优势 Strengths

集四家之长：Claude Code 的深度、Codex 的速度、Hermes 的进化、OpenCode 的开放。三重扩展体系（MCP + Hooks + Skill）业界最全。架构已预留多模型扩展能力，当前内置单一模型。管道模式与消息网关覆盖全场景。

W

劣势 Weaknesses

作为新进入者，品牌认知度低，社区生态需要时间积累。功能覆盖面广可能导致初期稳定性不如专注型竞品。未来开放多模型适配时需承担额外的测试与兼容性成本。

O

机会 Opportunities

AI Agent 市场仍处于早期，用户渴望一站式工具。企业对数据隐私敏感，本地优先架构有差异化优势。开源核心可快速获取开发者社区信任。MCP 协议成为事实标准，先发支持者受益。

T

威胁 Threats

OpenAI 与 Anthropic 资金雄厚，可能快速复制差异化功能。IDE 厂商（VS Code、JetBrains）原生集成 AI 可能压缩 CLI 工具空间。模型 API 价格波动影响云服务成本。

10 差异化卖点与功能取舍

10.1 核心差异化

  • ✓ 进化型 Agent — 唯一同时集成 GEPA 自进化循环与并行子代理编排的产品，执行重复任务时效率持续提升
  • ✓ 零锁定架构（架构就绪） — 模型层、工具层、记忆层完全解耦，工具层作为控制面标准化所有外部交互，为多模型切换与工具生态扩展预留接口
  • ✓ 全场景入口 — 同一 Agent 状态在终端、Web Dashboard、Slack、Telegram 间实时同步，满足移动办公需求
  • ✓ 安全优先 — 本地执行为主，云沙箱可选，所有外部调用通过工具控制面统一拦截与透明审计，支持域名白名单、速率限制与熔断，企业合规友好

10.2 明确不做（功能取舍）

  • ✗ 不替代 IDE — 专注终端场景，不与 VS Code、JetBrains 竞争编辑器市场，而是通过插件与之协作
  • ✗ 不做通用聊天 — 聚焦任务执行与代码工作流，不追求闲聊、创意写作等通用能力
  • ✗ 不托管代码 — 不构建 GitHub 替代方案，依赖用户现有的 Git 仓库与 CI/CD 流程
  • ✗ 不做闭源模型 — 不自研基础模型，专注 Agent 编排层与工具链，与所有顶尖模型合作

11 用户旅程

初次接触

用户通过 curl -fsSL https://nova-agent.dev/install.sh | bash 一键安装，nova setup 引导配置默认模型与 API Key，nova doctor 自检环境并给出优化建议。

首次任务

用户输入 nova 进入 REPL，描述第一个需求。Agent 展示生成的执行计划 DAG，用户确认后逐步执行，每步展示进度与中间结果。关键操作前暂停请求授权。

技能积累

完成 3-5 个同类型任务后，Agent 自动生成首个 Skill Document。后续类似任务执行速度提升 30%+。用户可 nova skills list 查看已习得技能。

生态扩展

用户发现需要连接内部数据库，通过 nova mcp add postgres-mcp 安装 MCP Server，Agent 立即获得 SQL 查询能力。配置 hooks.json 实现提交前自动通知 Slack。

深度依赖

Agent 成为日常开发标配：早会前自动 /goal 生成昨日代码审查摘要并推送 Slack，需求讨论时实时生成原型，故障排查时并行分析日志与代码，跨端状态同步让移动办公无中断，成为团队效率中枢。

12 典型使用场景

场景一：全栈功能开发

产品经理通过 Slack 发送需求：@nova 给用户中心加上双因素认证，要求支持 TOTP 和短信

Nova Agent 执行流程：
1. 分析现有 auth 模块结构（@file:src/auth/*）
2. 生成后端 API 设计（Node.js + Express）
3. 生成前端 UI 组件（React + Tailwind）
4. 编写单元测试与集成测试（Jest + Supertest）
5. 运行测试，失败则自动修复（3 轮迭代）
6. 生成数据库迁移脚本（Prisma）
7. 输出 CHANGELOG 与 API 文档更新
8. 代码审查引擎自动审查 diff，输出安全漏洞与性能建议
9. 创建 Git 分支并提交 commit
Total time: 4m 32s | Tests: 14 passed | Coverage: 91%

场景二：数据分析师的日报自动化

分析师每天上午运行：nova -q "拉取昨日订单数据，分析各渠道转化率，生成图表并发邮件给运营团队"

Nova Agent 执行流程：
1. MCP 连接内部 ClickHouse 数据库执行 SQL
2. Python 脚本清洗数据并计算转化漏斗
3. 生成 Matplotlib 图表（柱状图 + 趋势折线）
4. 渲染 HTML 邮件模板，嵌入图表 Base64
5. SMTP 发送邮件至运营组
6. 将 Skill 归档为 "daily-report"，次日自动优化 SQL
Total time: 1m 15s | Skill learned: daily-report v1

场景三：遗留系统迁移

技术负责人启动长期任务：nova /goal "将后端从 Express + JavaScript 迁移到 NestJS + TypeScript，保留所有 API 契约"

Nova Agent 执行流程（跨 3 天，异步后台执行）：
Day 1: 分析 127 个路由文件，生成类型定义，创建 NestJS 骨架
Day 2: 逐个迁移 Controller 与 Service，运行回归测试
Day 3: 清理旧代码，生成迁移指南，输出性能对比报告
用户随时通过 nova dashboard 查看进度，关键决策点推送 Slack 确认
Total time: 18h 42m | Files changed: 89 | Tests: 203 passed

场景四：运维故障排查

告警触发后，运维工程师运行：cat /var/log/app/error.log | nova "定位根因并给出修复建议"

Nova Agent 执行流程：
